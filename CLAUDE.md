# tg_userbot — Telegram Userbot с AI

## Обзор проекта и назначение

Telegram userbot на базе Telethon, который:
- Мониторит указанную группу и отвечает на сообщения пользователей из whitelist
- Самостоятельно инициирует разговоры на случайные темы из файла (сессии по 30 минут)
- Использует Gemini AI для генерации естественных ответов
- Хранит историю диалогов в SQLite для контекстных ответов

## Tech Stack

| Компонент    | Библиотека            | Назначение                          |
|--------------|-----------------------|-------------------------------------|
| MTProto      | telethon              | Подключение к Telegram как юзер     |
| AI           | google-generativeai   | Генерация ответов через Gemini API  |
| Scheduler    | apscheduler           | Планирование разговоров             |
| Database     | aiosqlite             | Асинхронное хранение истории        |
| Config       | pydantic-settings     | Загрузка настроек из .env           |
| Testing      | pytest, pytest-asyncio| TDD, асинхронные тесты              |
| Python       | 3.11+                 | Версия интерпретатора               |

## Архитектура (слои)

```
Корень проекта
├── ai/               ← Всё связанное с ИИ
│   ├── gemini.py     — Клиент Gemini API + загрузчик промтов из .md
│   ├── history.py    — Хранение истории диалогов в SQLite
│   └── prompts/      — Промты в формате .md (загружаются в runtime)
├── userbot/          ← Всё связанное с Telegram
│   ├── client.py     — Инициализация Telethon клиента
│   ├── handlers.py   — Обработчики сообщений, фильтр whitelist
│   └── scheduler.py  — APScheduler задачи, логика 30-мин сессий
├── core/             ← Общие утилиты
│   └── config.py     — Настройки через pydantic-settings
├── data/             ← Данные и ресурсы
│   ├── whitelist.md  — Список Telegram user_id (один на строку)
│   ├── topics.md     — Список тем для разговора
└── tests/            ← Тесты
```

### Поток данных
```
Входящее сообщение
  → handlers.py: проверка whitelist
  → history.py: загрузка истории
  → gemini.py: загрузка промта + генерация ответа
  → history.py: сохранение ответа
  → Отправка ответа

Таймер APScheduler
  → scheduler.py: выбор случайной темы из topics.md
  → gemini.py: генерация начала разговора
  → Отправка сообщения в группу
  → Сессия активна 30 минут
```

## TDD Правила для этого проекта

1. **Тест сначала** — писать тест до реализации модуля
2. **Красный → Зелёный → Рефакторинг** — классический цикл TDD
3. **Минимальная реализация** — stub делает ровно столько, чтобы тест прошёл
4. **Изоляция** — тесты не зависят от внешних сервисов:
   - Gemini API: мокировать через `unittest.mock`
   - SQLite: использовать `":memory:"` базу данных
   - Файлы whitelist/topics: `tempfile.NamedTemporaryFile`
5. **Асинхронные тесты** — все async тесты через `pytest-asyncio` с `asyncio_mode = "auto"`
6. **Русские docstrings** — все комментарии и docstrings строго на русском языке
7. **Запуск тестов**: `pytest` или `pytest --collect-only` для проверки сбора

## Ограничения

- **async везде** — все функции работы с БД и сетью только через `async def`
- **Только aiosqlite** — никаких синхронных вызовов SQLite
- **Без FastAPI** — нет HTTP-сервера, только Telegram userbot
- **Только SQLite** — никаких PostgreSQL, Redis и других СУБД
- **Промты из файлов** — промты не хардкодятся в коде, загружаются из `ai/prompts/*.md` в runtime
- **Whitelist из файла** — список пользователей загружается из `data/whitelist.md`
- **Темы из файла** — список тем загружается из `ai/prompts/topics.md`
- **Строковая сессия** — использовать `SESSION_STRING` из `.env`
- **Секреты** — `SESSION_STRING` не публиковать и не логировать
